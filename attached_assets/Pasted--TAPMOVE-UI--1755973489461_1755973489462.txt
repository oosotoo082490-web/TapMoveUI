탭무브(TAPMOVE) 공식 웹사이트를 제작해주세요.
목표: 화이트 기반의 고급·직관 디자인, 전면 한글 UI.
핵심: 사이트 내 세미나 신청, 로그인+후기 작성(세미나 참석자 전용 4자리 코드 게이트), 제품 결제(토스), 대량구매(수료자 전용 4자리 코드 게이트).

0) 전반

반응형(모바일/태블릿/데스크톱) 완벽 대응.

컬러: 화이트 배경 + 고대비 텍스트, 여백 넉넉, 카드 소프트 섀도우, rounded‑2xl.

상단 고정 내비(한국어): 홈 / 세미나 일정 / 신청 / 후기 / 센터 위치 / 로그인.

1) 홈 히어로(옵션 C) – 세로 영상

히어로 상단에 세로 비디오(mp4) 배치: 자동 재생, 반복(loop), 음소거(muted) 기본.

우측 하단에 작은 스피커 아이콘 토글: 클릭 시 음소거 해제/재적용.

상태는 세션 범위에서 기억(리로드 시 마지막 선택 유지 가능하면 적용).

영상 위에 간결 카피/버튼 두 개:

“세미나 신청하기(→ /apply)”

“세미나 일정 보기(→ /seminar)”

접근성: 키보드 포커스 가능, 아이콘에 대체 텍스트(“소리 켜기/끄기”).

2) 히어로 하단 섹션(옵션 A) – 가로 슬라이드 카드

가로 슬라이드 3장(반복 루프):

기본 자동재생: 2초마다 다음 장으로 이동

하단 중앙에 흰색 반투명 점 3개(동그라미) 페이지네이션:

사용자가 점을 클릭하면 해당 슬라이드로 즉시 이동

그 시점부터 3초 대기 후 다시 자동재생 재개(이후 2초 간격)

슬라이드는 스와이프/드래그 지원(모바일).

각 카드에는 탭무브 핵심 장점 3가지(짧은 문구 + 아이콘/썸네일).

3) 필수 페이지/기능

/seminar (세미나 일정 안내)

정보 카드: 일시, 교육시간, 접수마감, 교육비, 입금계좌: IM뱅크(구 대구은행) 50811‑8677704, 대상, 준비물, 장소(대구시 북구 침산남로 172, 3층).

참가혜택(자택 발급 문구 제외)

✅ 탭무브 공식 수료증 발급

✅ 탭무브 상표 사용 권한

✅ 세미나에서 배운 그대로, 수업에 바로 적용

✅ SNS/홈트에서 ‘TAPMOVE’ 명칭 공식 사용(수료자 한정)

상단/하단 고정 CTA: “지금 신청하기(→ /apply)”.

/apply (세미나 신청 폼)

필드(필수): 성명, 생년월일(YYYY‑MM‑DD), 이메일, 연락처, 주소, 입금자명.

버튼 그룹(단일 선택): 유니폼 사이즈 S/M/L/XL/XXL, 수업 진행 여부(“진행 예정” / “하지 않음”).

개인정보 동의 체크(필수) + 모달 약관.

제출 후 성공 화면: 접수번호/시간, 계좌/문의 안내.

알림(비용 절감 우선)

기본: 관리자 대시보드 실시간 갱신(웹소켓 또는 간단 폴링).

옵션: SMTP ENV가 있으면 이메일 알림(관리자 수신). 내용은 비용 최소화를 위해 “세미나 신청이 접수되었습니다.” + 이름/연락처만 포함.

SMS는 유료 과금이므로 기본 미사용(주석으로 확장 포인트만 남김).

/reviews (후기 — 세미나 참석자 전용 게이트)

상단 “후기 작성하기” 클릭 시 모달:
“세미나 참석자만 작성 가능합니다. 대표자에게 안내받은 비밀번호 4자리를 입력해주세요.”

입력: 숫자 4자리만 허용.

서버의 ENV/DB에 저장된 해시(reviewPasscodeHash) 와 일치하면 세션 플래그(canWriteReview=true) 설정 → 후기 작성 페이지 이동.

후기 작성: 텍스트, 별점(1–5), 이미지 1장(2MB 제한). 제출 시 관리자 승인 후 공개.

목록: 정렬/필터(최신/별점/태그), 신고→관리자 숨김/삭제.

/location (센터 위치)

텍스트 주소/안내만으로 충분히 보기 좋게 구성.

비용 없는 경우에만 오픈스트리트맵(Leaflet) 임베드 토글 제공(기본은 텍스트만). 출처 표기.

/product (제품 소개/구매)

정식 로고 6mm 매트 소개/사진.

일반 구매: 장바구니/결제(아래 결제=토스).

대량구매 버튼: 클릭 시 세미나 수료자 전용 4자리 비밀번호 모달(문구 동일, 필드 bulkPasscode). 검증 통과 시 대량구매 폼 표시:

수량: 최소 20개, 선택은 20부터 10개 단위(20,30,40,50,…) 무한 증가

단가×수량 실시간 합계 표시

장바구니/결제 플로우 동일.

/policy (정책)

경고 박스(상표·수업 사용 정책):
“✅ TAPMOVE는 상표등록된 지식재산입니다. 정식 매트 외 유사 제품·수업·명칭 도용은 민·형사상 책임 대상이며, 정식 세미나 참석자만 공식 사용 가능합니다.”

4) 인증/권한(비용 절감)

로컬 이메일+비밀번호(bcrypt), 세션/쿠키, CSRF.

관리자 페이지(/admin)는 숨김 처리: 주소를 직접 입력해도 로그인 상태·역할이 admin일 때만 접근/노출.

Applications(검색/필터/상태 변경/CSV)

Reviews(승인/숨김/삭제, 신고 처리)

Products(단가/이미지/재고)

Orders(결제 상태/상세)

Settings: 세미나 일정·계좌·마감, 4자리 코드 설정 UI(리뷰용/대량구매용 각각 입력→서버에서 해시 저장), SMTP/결제 키.

🔧 후기(Reviews) 기능 수정 지시 — 텍스트 전용 + 자동 욕설 필터 + 비공개 처리

요구 요약(한글 UI 고정):

후기 = 텍스트만 작성 (별점/이미지 입력 제거).

자동 욕설/스팸 필터링: 제출 시 서버가 텍스트를 검사하여 부적절 표현이 감지되면, 저장은 하되 공개 목록에서는 자동 숨김 처리.

상태값 예: status='hidden_by_filter'.

관리자는 /admin에서 모든 후기(정상/필터됨/신고됨/숨김) 를 볼 수 있음. 일반 사용자는 status='published' 만 보임.

제출 UX: 사용자가 욕설을 써도 “완료되었습니다” 안내 또는 기본 리다이렉트로 정상 제출 플로우를 유지(필터링 사실은 노출하지 않음).

제출 후 POST /reviews → 302 Redirect → /reviews?submitted=1 로 이동(페이지가 한 번 깜빡이며 새로고침).

이때 일반 사용자는 방금 쓴 욕설 후기를 포함해 필터된 후기는 목록에서 보이지 않음.

관리자 페이지: hidden_by_filter 포함 모든 상태 열람/검색/필터 가능.

관리자 조치: “공개 전환(publish) / 유지(hidden) / 삭제(delete)” 버튼 제공.

리스트에 상태 배지(예: Published / Hidden by Filter / Pending / Reported) 표시.

상세 구현 지시:

후기 작성 권한은 기존과 동일: 세미나 참석자 전용 4자리 코드 게이트 통과 세션(canWriteReview=true)일 때만 작성 가능.

후기 모델 스키마(기존에서 수정):

Review(id, userId, body TEXT, status ENUM['published','hidden_by_filter','hidden','pending'], createdAt)

별점, 이미지 필드 제거.

서버 측 필터(한국어 욕설/비속어/광고 패턴):

기본: 내장 욕설 사전 + 정규식(띄어쓰기/특수문자/자음모음 분리/의도적 철자 변형 일부 대응).

예시: 대표적인 한국어 비속어/광고 유도(전화/링크 스팸) 패턴을 포함하고, 단어 경계 무시 토큰화(공백/특수문자 삽입)도 최소한 대응.

매칭되면 status='hidden_by_filter' 로 저장.

매칭되지 않으면 기본 status='published'.

목록 쿼리:

일반 사용자: WHERE status='published' 만 반환.

관리자: 상태 제한 없이 전부 반환(필터/검색 UI 제공).

신고(Report) 기능은 유지: 사용자가 신고하면 관리자가 상태를 hidden 으로 수동 전환 가능.

감사 로그(Optional): ModerationLog(reviewId, adminId, action, createdAt) 남기면 좋음(선택).

레이트 리밋: 후기 제출 API에 IP/계정 기준 rate limit 적용(스팸 방지).

길이 제한: 본문 최소/최대 길이 검증(예: 10자 ~ 2,000자).

UI/카피(한글 고정):

후기 작성 버튼: “후기 작성하기”

4자리 코드 모달: “세미나 참석자만 작성 가능합니다. 대표자에게 안내받은 비밀번호 4자리를 입력해주세요.”

제출 성공 후(공용): /reviews?submitted=1 로 리다이렉트, 상단에 토스트 또는 배너: “후기가 접수되었습니다.”

필터 여부는 사용자에게 공개하지 않음.

관리자 리뷰 리스트: 상태 배지와 함께 정렬/검색/필터(상태별) 제공.

액션: “공개 전환”, “숨김 유지”, “삭제” 버튼.

테스트 시나리오:

일반 텍스트 후기 → published 로 노출(제출 후 새로고침, 목록에서 보임).

욕설 포함 후기 → 저장되지만 hidden_by_filter 로 처리되어 사용자 목록에 미노출, 관리자 화면에서만 보임.

제출 직후 페이지 새로고침되어도 2번 케이스는 작성자 포함 누구에게도 비공개(관리자만 열람).

관리자가 hidden_by_filter 항목을 “공개 전환”하면 즉시 사용자 목록에 반영.

위 사양으로 전체 프로젝트를 스캐폴딩하고 구현해주세요.
